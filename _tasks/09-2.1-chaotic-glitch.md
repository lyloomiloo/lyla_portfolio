# Task: Chaotic scramble-to-land entrance animation
Plan: `/_plans/09-visual-overhaul.md` → Step 2.1
Spec: `/_specs/09-visual-overhaul.md`

## Context
The current glitch is too clean — single swap, clean restore. It should feel like characters are being decoded from noise, like a transmission resolving. The entrance should be especially dramatic.

## What to build
Rewrite the GlitchText JS to add a scramble-to-land entrance and make periodic glitch more chaotic.

## Technical details

### Scramble-to-land entrance (when playEntrance=true)
1. On mount, fill ALL character positions with random glitch symbols
2. Cycle these symbols rapidly (new random char every 40-60ms per position)
3. Characters "land" (lock to their real value) one-by-one from left to right
4. Gap between each character landing: random 30-80ms
5. Total entrance duration for "LYLA HUANG." (~10 chars): ~500-800ms
6. Non-letter chars (spaces, periods, line breaks) land immediately
7. Replace the CSS clip-path `glitchIn` animation — this JS decode replaces it entirely

### Multi-flicker periodic glitch
Instead of: swap → wait 100ms → restore
Do: swap → wait 40ms → partial-restore (different random chars) → wait 40ms → swap again → wait 40ms → full restore
This creates a rapid flicker effect (3 frames of visual noise before settling).

### Glitch characters
Keep: `* # @ / \ ] [ 0 1 { } < > _ | ~ $ % &`

### Props (unchanged)
- `text`, `tag`, `class`, `intensity`, `interval`, `playEntrance`

## Files to create/modify
- `src/components/GlitchText.astro` — rewrite script section

## Acceptance criteria
- Hero title scrambles from noise and lands character by character on page load
- Landing moves left-to-right with visible stagger
- Periodic glitch has 2-3 rapid flickers before restoring
- Other GlitchText instances (project titles, section labels) only do periodic glitch (no entrance)
- No layout shift during any animation
