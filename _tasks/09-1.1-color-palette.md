# Task: Update global palette to purple + yellow complementary
Plan: `/_plans/09-visual-overhaul.md` → Step 1.1
Spec: `/_specs/09-visual-overhaul.md`

## Context
The current black+blue+yellow palette uses direct contrast. Switching to purple + yellow (complementary on the color wheel) creates vibrant visual tension that feels more intentional and edgy. No pure black — use dark charcoal grey instead.

## What to build
Update every color reference across the entire codebase.

## Technical details

### New CSS variables in global.css
```css
:root {
  --purple: #7B2DFF;      /* replaces --blue as primary accent */
  --yellow: #FFE600;       /* stays */
  --dark: #2A2A2A;         /* replaces --black, never use #000 */
  --dark-light: #3A3A3A;   /* for borders on dark backgrounds */
}
```

### Removals / renames
- `--blue: #0038FF` → remove, replace all uses with `var(--purple)`
- `--black: #000000` → remove, replace all uses with `var(--dark)`
- `--red`, `--pink`, `--green` → keep in CSS vars but don't use as primary. Award type color can stay subtle.

### Files to update (every instance of var(--blue), var(--black), #000, #0038FF)
- `global.css` — add --purple, --dark, --dark-light; remove --black
- `Hero.astro` — accent dot color
- `GlitchText.astro` — .accent-dot color
- `ProjectCard.astro` — mockup bg, link color
- `RetroElements.astro` — all black bgs → dark, blue text → purple, robot colors
- `SkillsSection.astro` — category pill colors
- `SignalLog.astro` — label/highlight colors, border
- `Contact.astro` — border, status color, hover
- `AboutSection.astro` — pullquote border
- `Nav.astro` — no changes needed (uses --text/--text-dim)

## Acceptance criteria
- No `#000000` or `#000` anywhere in CSS
- No `var(--blue)` or `var(--black)` references remain
- Purple (#7B2DFF) is the primary accent color
- Yellow (#FFE600) is the secondary accent color
- Dark strips use #2A2A2A, not pure black
